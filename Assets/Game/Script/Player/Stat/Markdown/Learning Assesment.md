# L4: การออกแบบการประเมินผลการเรียนรู้  
*(Learning Assessment Design)*

ทุกครั้งที่ผู้เล่นจบ **ภารกิจการสำรวจ (Expedition)** ซึ่งหมายถึงการปีนเขาแต่ละครั้ง  
ระบบจะสรุปผลการปฏิบัติงาน (**Performance Report**) เพื่อสะท้อนความสามารถของผู้เล่นในด้านต่าง ๆ  
โดยแบ่งเกณฑ์การประเมินออกเป็น 3 ด้านหลัก พร้อมสัดส่วนคะแนนดังนี้

---

## ตารางสัดส่วนคะแนนรวม

| หมวดการประเมิน | รายละเอียด | สัดส่วน (%) |
|----------------|------------|-------------|
| **1. ประสิทธิภาพ (Efficiency)** | การใช้ทรัพยากร เช่น พลังงาน น้ำ อาหาร อย่างเหมาะสม ไม่สิ้นเปลือง | 40% |
| **2. ความปลอดภัย (Safety)** | การหลีกเลี่ยงสถานการณ์อันตราย เช่น ทางชัน พายุ หินถล่ม | 30% |
| **3. การวางแผนและการตัดสินใจ (Planning & Decision-Making)** | การเลือกเส้นทาง จุดพัก และการปรับแผนเมื่อเกิดเหตุไม่คาดคิด | 30% |

## ระบบคำนวณคะแนน

### 1. ด้านประสิทธิภาพ (Efficiency – 40%)

**เป้าหมาย**  
ใช้ทรัพยากรให้เหมาะสมกับระยะทางและความชัน โดยวัดจาก:
- **Stamina** (พลังงานที่ใช้ไป)
- **Food Items** (จำนวนอาหารที่กินไป)
- **Water Items** (จำนวนน้ำที่ดื่มไป)

**วิธีคำนวณ**

1. คำนวณอัตราส่วนการใช้ทรัพยากรแต่ละประเภท:
```
Stamina Ratio = Stamina ที่ใช้จริง / Stamina ที่เหมาะสม
Food Ratio = จำนวนอาหารที่กิน / จำนวนอาหารที่เหมาะสม
Water Ratio = จำนวนน้ำที่ดื่ม / จำนวนน้ำที่เหมาะสม
```

2. หาค่าเฉลี่ยของอัตราส่วนทั้ง 3:
```
Average Ratio = (Stamina Ratio + Food Ratio + Water Ratio) / 3
```

3. คำนวณคะแนนประสิทธิภาพ:
```
Efficiency Score = 100 - ((Average Ratio - 1) × 100)
```

**ตัวอย่างการคำนวณ**

**สถานการณ์ที่ 1: ใช้ทรัพยากรใกล้เคียงที่เหมาะสม**
- Stamina: ใช้ 520 จาก 500 ที่เหมาะสม → Ratio = 1.04
- Food: กิน 6 ชิ้น จาก 5 ชิ้นที่เหมาะสม → Ratio = 1.20
- Water: ดื่ม 8 ชิ้น จาก 7 ชิ้นที่เหมาะสม → Ratio = 1.14
- Average Ratio = (1.04 + 1.20 + 1.14) / 3 = **1.13**
- Efficiency Score = 100 - ((1.13 - 1) × 100) = 100 - 13 = **87**
- คะแนนถ่วงน้ำหนัก = 87 × 0.4 = **34.8**

**สถานการณ์ที่ 2: ใช้ทรัพยากรมากเกินไป**
- Stamina: ใช้ 800 จาก 500 ที่เหมาะสม → Ratio = 1.60
- Food: กิน 10 ชิ้น จาก 5 ชิ้นที่เหมาะสม → Ratio = 2.00
- Water: ดื่ม 12 ชิ้น จาก 7 ชิ้นที่เหมาะสม → Ratio = 1.71
- Average Ratio = (1.60 + 2.00 + 1.71) / 3 = **1.77**
- Efficiency Score = 100 - ((1.77 - 1) × 100) = 100 - 77 = **23**
- คะแนนถ่วงน้ำหนัก = 23 × 0.4 = **9.2**

**การตีความคะแนน**

| คะแนนที่ได้ | การใช้ทรัพยากร | ความหมาย |
|------------|----------------|----------|
| 90–100 | ≤ 110% | ดีเยี่ยม - ใช้ทรัพยากรอย่างมีประสิทธิภาพสูง |
| 70–89 | 111–130% | ดี - ใช้ทรัพยากรเกินไปเล็กน้อย |
| 50–69 | 131–160% | พอใช้ - ใช้ทรัพยากรมากเกินไป ควรปรับปรุง |
| 0–49 | >160% | ควรปรับปรุง - สิ้นเปลืองมาก ขาดการวางแผน |

### 2. ด้านความปลอดภัย (Safety – 30%)

**เป้าหมาย**  
หลีกเลี่ยงสถานการณ์อันตรายและลดการบาดเจ็บให้น้อยที่สุด

**ประเภทความเสี่ยงที่ตรวจจับได้:**
- 🏔️ ทางลาดชัน (Steep Slope)
- ⛈️ อันตรายจากสภาพอากาศ (Weather Hazard)
- 🪨 หินถล่ม (Rockfall)
- 🧊 พื้นน้ำแข็ง (Ice Patch)
- ❄️ หิมะถล่ม (Avalanche)
- 💤 อ่อนเพลีย (Exhaustion)

**วิธีคำนวณ**

1. คำนวณอัตราการหลีกเลี่ยงอันตราย:
```
Avoidance Rate = 1 - (จำนวนเหตุการณ์เสี่ยงที่พบ / เหตุการณ์เสี่ยงทั้งหมด)
```

2. คำนวณคะแนนพื้นฐานจากการหลีกเลี่ยง:
```
Base Score = Avoidance Rate × 100
```

3. หักคะแนนจากการบาดเจ็บ:
```
Health Penalty = จำนวนครั้งที่เสียเลือด × 5
```

4. คำนวณคะแนนความปลอดภัยสุดท้าย:
```
Safety Score = Base Score - Health Penalty
(คะแนนต่ำสุด 0, สูงสุด 100)
```

**ตัวอย่างการคำนวณ**

**สถานการณ์ที่ 1: ระมัดระวังสูง ไม่บาดเจ็บ**
- เหตุการณ์เสี่ยงทั้งหมด: 10 จุด
- พบเหตุการณ์เสี่ยง: 2 ครั้ง (หลีกเลี่ยงได้ 8 ครั้ง)
- จำนวนครั้งที่เสียเลือด: 0 ครั้ง

การคำนวณ:
- Avoidance Rate = 1 - (2/10) = 0.80
- Base Score = 0.80 × 100 = 80
- Health Penalty = 0 × 5 = 0
- Safety Score = 80 - 0 = **80**
- คะแนนถ่วงน้ำหนัก = 80 × 0.3 = **24.0**

**สถานการณ์ที่ 2: หลีกเลี่ยงได้บางส่วน มีการบาดเจ็บ**
- เหตุการณ์เสี่ยงทั้งหมด: 10 จุด
- พบเหตุการณ์เสี่ยง: 5 ครั้ง (หลีกเลี่ยงได้ 5 ครั้ง)
- จำนวนครั้งที่เสียเลือด: 3 ครั้ง

การคำนวณ:
- Avoidance Rate = 1 - (5/10) = 0.50
- Base Score = 0.50 × 100 = 50
- Health Penalty = 3 × 5 = 15
- Safety Score = 50 - 15 = **35**
- คะแนนถ่วงน้ำหนัก = 35 × 0.3 = **10.5**

**สถานการณ์ที่ 3: ปลอดภัยสูงสุด**
- เหตุการณ์เสี่ยงทั้งหมด: 10 จุด
- พบเหตุการณ์เสี่ยง: 0 ครั้ง (หลีกเลี่ยงได้ทั้งหมด)
- จำนวนครั้งที่เสียเลือด: 0 ครั้ง

การคำนวณ:
- Avoidance Rate = 1 - (0/10) = 1.00
- Base Score = 1.00 × 100 = 100
- Health Penalty = 0 × 5 = 0
- Safety Score = 100 - 0 = **100**
- คะแนนถ่วงน้ำหนัก = 100 × 0.3 = **30.0**

**การตีความคะแนน**

| คะแนนที่ได้ | อัตราหลีกเลี่ยง | ความหมาย |
|------------|----------------|----------|
| 90–100 | ≥ 90% | ดีเยี่ยม - ปลอดภัยมาก หลีกเลี่ยงอันตรายได้เกือบทั้งหมด |
| 70–89 | 70–89% | ดี - ปลอดภัย แต่ยังมีความเสี่ยงบางส่วน |
| 50–69 | 50–69% | พอใช้ - เสี่ยงค่อนข้างมาก ควรระมัดระวังมากขึ้น |
| 0–49 | <50% | อันตราย! - พบเหตุการณ์เสี่ยงและบาดเจ็บมากเกินไป |

### 3. ด้านการวางแผนและการตัดสินใจ (Planning – 30%)

**เป้าหมาย**  
เลือกเส้นทางที่มีประสิทธิภาพสูงสุด โดยสมดุลระหว่าง:
- 🗺️ **ระยะทาง** (Distance) - 50%
- ⏱️ **เวลา** (Time) - 30%
- ⚡ **พลังงาน** (Stamina) - 20%

**วิธีคำนวณ**

1. คำนวณต้นทุนเส้นทางที่ใช้จริง (Actual Path Cost):
```
Actual Cost = (Distance × 0.5) + (Time × 0.3) + (Stamina × 0.2)
```

2. คำนวณต้นทุนเส้นทางที่เหมาะสม (Optimal Path Cost):
```
Optimal Cost = (Optimal Distance × 0.5) + (Optimal Time × 0.3) + (Optimal Stamina × 0.2)
```

3. คำนวณค่าเบี่ยงเบน (Deviation):
```
Deviation = |(Actual Cost - Optimal Cost) / Optimal Cost|
```

4. คำนวณคะแนนการวางแผน:
```
Planning Score = 100 - (Deviation × 100)
(คะแนนต่ำสุด 0, สูงสุด 100)
```

**ตัวอย่างการคำนวณ**

**สถานการณ์ที่ 1: เส้นทางใกล้เคียงที่สุด**

**เส้นทางที่เหมาะสม (Optimal):**
- Distance: 1200 เมตร
- Time: 480 วินาที (8 นาที)
- Stamina: 500 หน่วย

**เส้นทางที่ใช้จริง (Actual):**
- Distance: 1250 เมตร
- Time: 510 วินาที (8.5 นาที)
- Stamina: 520 หน่วย

การคำนวณ:
```
Optimal Cost = (1200 × 0.5) + (480 × 0.3) + (500 × 0.2)
             = 600 + 144 + 100
             = 844

Actual Cost = (1250 × 0.5) + (510 × 0.3) + (520 × 0.2)
            = 625 + 153 + 104
            = 882

Deviation = |882 - 844| / 844
          = 38 / 844
          = 0.045 (4.5%)

Planning Score = 100 - (0.045 × 100)
               = 100 - 4.5
               = 95.5
```
- คะแนนถ่วงน้ำหนัก = 95.5 × 0.3 = **28.65**

**สถานการณ์ที่ 2: เส้นทางอ้อมไกล**

**เส้นทางที่เหมาะสม (Optimal):**
- Distance: 1200 เมตร
- Time: 480 วินาที
- Stamina: 500 หน่วย

**เส้นทางที่ใช้จริง (Actual):**
- Distance: 1800 เมตร (อ้อมไกล)
- Time: 720 วินาที (12 นาที)
- Stamina: 800 หน่วย

การคำนวณ:
```
Optimal Cost = (1200 × 0.5) + (480 × 0.3) + (500 × 0.2) = 844

Actual Cost = (1800 × 0.5) + (720 × 0.3) + (800 × 0.2)
            = 900 + 216 + 160
            = 1276

Deviation = |1276 - 844| / 844
          = 432 / 844
          = 0.512 (51.2%)

Planning Score = 100 - (0.512 × 100)
               = 100 - 51.2
               = 48.8 (clamp to 48.8)
```
- คะแนนถ่วงน้ำหนัก = 48.8 × 0.3 = **14.64**

**สถานการณ์ที่ 3: เส้นทางดีที่สุด**

**เส้นทางที่เหมาะสม (Optimal):**
- Distance: 1200 เมตร
- Time: 480 วินาที
- Stamina: 500 หน่วย

**เส้นทางที่ใช้จริง (Actual):**
- Distance: 1200 เมตร
- Time: 480 วินาที
- Stamina: 500 หน่วย

การคำนวณ:
```
Optimal Cost = 844
Actual Cost = 844
Deviation = 0
Planning Score = 100
```
- คะแนนถ่วงน้ำหนัก = 100 × 0.3 = **30.0**

**การตีความคะแนน**

| คะแนนที่ได้ | ค่าเบี่ยงเบน | ความหมาย |
|------------|-------------|----------|
| 90–100 | ≤ 10% | ดีเยี่ยม - เลือกเส้นทางที่เหมาะสมมาก |
| 70–89 | 11–30% | ดี - วางแผนดี แต่ยังมีเส้นทางที่ดีกว่า |
| 50–69 | 31–50% | พอใช้ - วางแผนพอใช้ ควรเลือกเส้นทางที่ดีกว่า |
| 0–49 | >50% | ควรปรับปรุง - วางแผนไม่ดี เลือกเส้นทางที่ไม่เหมาะสม |

## การคำนวณค่าที่เหมาะสม (Optimal Metrics)

ระบบจะคำนวณค่าที่เหมาะสมโดยอัตโนมัติหรือรับค่าจากระบบวางแผนเส้นทาง:

### วิธีที่ 1: คำนวณอัตโนมัติ (Auto-Calculate)

ระบบจะคำนวณจากระยะทางเส้นทาง:

```
Expected Stamina = Distance × 0.5
Expected Food Items = (Distance / 1000) × 2    // ~2 ชิ้นต่อกิโลเมตร
Expected Water Items = (Distance / 1000) × 3   // ~3 ชิ้นต่อกิโลเมตร
Optimal Time = Distance / 2.5                   // ความเร็ว 2.5 เมตร/วินาที
```

**ตัวอย่าง:** เส้นทาง 1500 เมตร
- Expected Stamina = 1500 × 0.5 = **750 หน่วย**
- Expected Food = (1500 / 1000) × 2 = **3 ชิ้น**
- Expected Water = (1500 / 1000) × 3 = **4.5 ชิ้น** → ปัดเป็น **5 ชิ้น**
- Optimal Time = 1500 / 2.5 = **600 วินาที** (10 นาที)

### วิธีที่ 2: ใช้ค่าจากระบบวางแผน (External Optimal)

ถ้ามีระบบวางแผนเส้นทาง สามารถกำหนดค่าที่เหมาะสมได้โดยตรง:
- ระบบวิเคราะห์ภูมิประเทศ (ความชัน สภาพอากาศ)
- คำนวณค่าที่เหมาะสมแม่นยำกว่า
- ส่งค่าให้ระบบประเมินใช้เปรียบเทียบ

## ระดับผลการประเมิน (Performance Rank Thresholds)

| คะแนนรวม | ระดับ | คำอธิบาย |
|---------|------|----------|
| 90–100 | 🏔️ Alpine Master | วางแผนและจัดการทรัพยากรได้สมบูรณ์แบบ |
| 70–89 | 🧭 Skilled Planner | วางแผนดี ใช้ทรัพยากรมีประสิทธิภาพ |
| 50–69 | ⚙️ Survivor | เอาตัวรอดได้แต่ยังขาดการวางแผน |
| 0–49 | 🪶 Lost Wanderer | ใช้ทรัพยากรสิ้นเปลือง ขาดการวางแผน |

## สรุป

ระบบประเมินผลการเรียนรู้ (Learning Assessment System) ช่วยให้ผู้เล่น:

✅ **เข้าใจ** จุดแข็งและจุดอ่อนของตนเอง  
✅ **เรียนรู้** วิธีจัดการทรัพยากรอย่างมีประสิทธิภาพ  
✅ **พัฒนา** ทักษะการวางแผนและการตัดสินใจ  
✅ **หลีกเลี่ยง** อันตรายและสถานการณ์เสี่ยง  
✅ **ปรับปรุง** ผลการเล่นในครั้งถัดไป  

**การคำนวณ:** ระบบใช้สูตรถ่วงน้ำหนัก 40:30:30 (Efficiency:Safety:Planning) พร้อมเกณฑ์การให้คะแนนที่ชัดเจนและยุติธรรม

**ผลลัพธ์:** ผู้เล่นจะได้รับ Performance Rank พร้อมคำแนะนำเป็นภาษาไทยเพื่อพัฒนาทักษะ
